# Global Git Workflow Rules

Conventions for branch naming, commits, and pull requests.

---

## 1. Branch Naming Rules
- Use feature-scoped prefixes:
  - `feat/feature-name` (e.g. `feat/finance-claims`)
  - `fix/bug-description` (e.g. `fix/auth-cookie-expire`)
  - `docs/topic-name` (e.g. `docs/aios-setup`)
  - `refactor/refactor-goal` (e.g. `refactor/bookings-calendar`)

## 2. Commit Message Standards
- Use the imperative mood in commit summaries (e.g. `Add dynamic leave balance verification`).
- Keep commit titles under 50 characters.
- Link relevant issues or tasks if available.

## 3. Pull Request Guidelines
- Always verify compilation and format checks locally before submitting.
- Document visual layout modifications with screenshot attachments.
