# ADR-002: Stateless jose JWT Authentication

* **Status**: Accepted
* **Date**: 2026-07-14

## Context
Secure user authentication is required across both Next.js web API paths and embedded mobile WebViews.

## Problem
Maintaining a database-backed sessions table adds significant DB reads overhead and latency to every request.

## Options
1. Database-backed sessions (e.g. NextAuth default adapters).
2. Stateless JWT tokens stored in HTTP-only cookies, verified via the `jose` library.

## Decision
Choose Option 2: Implement token-based stateless authentication using JWTs managed by the `jose` library.

## Consequences
- Prevents database operations overhead on verification lookups.
- Enables direct cookie validation within Next.js API routes and mobile auth handoff.
