# ADR-001: SQLite Dev, PostgreSQL Prod

* **Status**: Accepted
* **Date**: 2026-07-14

## Context
We need a database layout that supports fast local development setups, rapid iteration, and highly concurrent production usage.

## Problem
Running heavy PostgreSQL databases locally adds setup friction for developers. However, SQLite lacks robust Row-Level Security (RLS) and scaling constraints needed for production deployment.

## Options
1. PostgreSQL for both dev and production.
2. SQLite for dev and PostgreSQL for production, managed by Drizzle ORM.

## Decision
Choose Option 2: SQLite (`dev.db`) for development environment and PostgreSQL for production. Drizzle ORM allows seamless dual-dialect switching.

## Consequences
- Developers can spin up the workspace instantly with zero external DB configuration.
- Production environment preserves PostgreSQL security and performance.
