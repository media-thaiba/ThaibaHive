# Product

## Register

product

## Users

All staff (800–1,000 employees) across Thaiba Garden Group of Institutions — teachers, administrative staff, department heads (HODs), and super admins. Most use the web app from a desktop during work hours; some access via tablet for attendance marking. Not all are tech-savvy, so clarity and predictability matter more than novelty.

## Product Purpose

ThaibaHive is the unified staff management platform for the institution group. It replaces spreadsheets, paper registers, and ad-hoc messaging with a single place for attendance, leaves, tasks, reports, announcements, bookings, and IT support. Success looks like every staff member using it daily without training.

## Brand Personality

- **Trustworthy but warm** — precision of a tool, friendliness of a colleague
- **Quietly capable** — it works without needing to prove it works
- **Institution-appropriate but not institutional** — serious enough for HR compliance, comfortable enough for daily use

## Anti-references

The three buckets to avoid entirely:
1. **ERP gray** — dense tables, muted everything, no hierarchy, feels like punishment
2. **Bootcamp shadcn** — default card grid, unstyled selects, placeholder gradients, "I ran npx create-app"
3. **Cartoon admin** — oversized icons, bright primary colors, childish typography, clip-art illustrations

None of the three above. The interface should feel custom-built for this institution, not templated.

## Design Principles

1. **Reduce decisions** — every screen should surface the one thing the user needs to do next. Hide configuration behind the things that need configuring.
2. **Information scent** — labels, headings, and links should tell the user what they'll find. No vague "Manage" buttons.
3. **Visual rhythm** — consistent spacing, balanced content density, intentional whitespace. Cramming is not efficiency.
4. **Feedback before failure** — show success/error states inline, not in toasts that disappear. Let the user know the system heard them.
5. **Device-respecting** — desktop-first but not desktop-only. Forms and tables should survive tablet widths without horizontal scroll.

## Accessibility & Inclusion

- Aim for WCAG 2.1 AA
- Reduced motion: all animations degrade gracefully
- Color is never the sole carrier of information (status badges have text labels)
- Forms use proper labels, not placeholders as labels
